Git

Hoy en día, Git es, con diferencia, el sistema de control de versiones moderno más utilizado del mundo. Git es un proyecto de código abierto maduro y con un mantenimiento activo que desarrolló originalmente Linus Torvalds, el famoso creador del kernel del sistema operativo Linux, en 2005. Un asombroso número de proyectos de software dependen de Git para el control de versiones, incluidos proyectos comerciales y de código abierto. Los desarrolladores que han trabajado con Git cuentan con una buena representación en la base de talentos disponibles para el desarrollo de software, y este sistema funciona a la perfección en una amplia variedad de sistemas operativos e IDE (entornos de desarrollo integrados).

Cuando implementamos git en un proyecto web, hablamos de crear un repositorio de nuestro código, el cual puede ser publicada, llevar un control de cambios y también podemos clonar (o copiar) un repositorio ajeno que sea público.

Comandos básicos

El siguiente listado de comandos pueden ser ejecutados por medio de la consola de comandos ubicados dentro del proyecto web.

Iniciar un repositorio git

Crear un directorio o carpeta vacia, o simplemente empezar con un proyecto ya hecho.

git init

Agregar los archivos o cambios al repositorio

Un o varios archivos especificados

git add

Todos los archivos

git add .

Crear un commit, siempre después de un add

Se realiza siempre despues de git add y es para crear un marca, a la cual podemos regresar para ver el estado del código en es punto

git commit -m "Actualización de listas"
